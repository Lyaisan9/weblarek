# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

##### Данные
В ходе анализа проекта было установлено: в приложении используются две сущности, которые описывают данные, — товар и покупатель. Их можно описать такими интерфейсами:
Товар:
interface IProduct {
id: string; // идентификатор товара
description: string; // подробное описание товара
image: string; // URL-адрес изображения товара
title: string; // название товара
category: string; // категория, к которой относится товар
price: number | null; // цена товара
}
Интерфейс IProduct описывает товар, доступный для покупки в приложении.

Покупатель:
interface IBuyer {
payment: TPayment; // способ оплаты товара  
 email: string; // почта покупателя
phone: string; // телефонный номер покупателя
address: string; // адрес проживания покупателя
}
Интерфейс IBuyer описывает данные покупателя, необходимые для оформления заказа.

##### Модели данных

Для учёта данных в приложении должны быть три класса, которые будут разделены между собой по смыслу и зонам ответственности:

1. ProductCatalog - хранение товаров, которые можно купить в приложении;
2. ShoppingCart - хранение товаров, которые пользователь выбрал для покупки;
3. Buyer - данные покупателя, которые тот должен указать при оформлении заказа.

Класс ProductCatalog - это шаблон для создания «каталога товаров».

Назначение и зона ответственности:
Хранит полный список всех доступных товаров в системе (хранит массив всех товаров). Также отвечает за управление выделенным товаром (хранит товар, выбранный для подробного отображения)

Поля класса

1. products тип данных: IProduct[] (массив объектов, соответствующих интерфейсу IProduct).
   Что хранит: полный список всех товаров, доступных для покупки в приложении.
   За что отвечает: хранение каталога товаров, предоставление данных для отображения списка товаров.
2. selectedProduct хранит один выбранный товар IProduct или null, если ничего не выбрано.

Методы ProductCatalog:

setProducts(products: IProduct[]): void - сохранение массива товаров полученного в параметрах метода, параметры: products (IProduct[]) — массив товаров, возвращаемое значение: отсутствует (void).

getProducts(): IProduct[] - получение массива товаров из модели, параметр отсутствует, возвращаемое значение: IProduct[] — массив товаров.

getProductById(id:string): IProduct| null - получение одного товара по его id, параметр id, возвращаемое значение: IProduct | null — товар(весь объект) по id или null, если товар не выбран.

setSelectedProduct(product: IProduct): void - сохранение товара для подробного отображения, параметры: product (IProduct) — товар, который нужно отобразить, возвращаемое значение: отсутствует (void).

getSelectedProduct(): IProduct | null - получение товара для подробного отображения,параметры: отсутствуют, возвращаемое значение: IProduct | null — выбранный товар или null, если товар не выбран.

Класс ShoppingCart - шаблон для создания товаров для покупки.

Назначение и зона ответственности: хранит массив товаров, выбранных покупателем для покупки.

Поле класса: items: IProduct[] — массив товаров, добавленных в корзину.

Методы ShoppingCart:

getShoppingCart(): IProduct[] - получение массива товаров, которые находятся в корзине, параметр отсутствует, возвращаемое значение: IProduct[] — массив товаров в корзине.

addItem(product: IProduct): void - добавление товара, который был получен в параметре, в массив корзины, параметр: product (IProduct) — товар, который нужно добавить, возвращаемое значение: отсутствует (void).

removeItem(product: IProduct): void - удаление товара, полученного в параметре из массива корзины, параметры: product (IProduct) — товар, который нужно удалить,возвращаемое значение: отсутствует (void).

clearItem(): void {} - очищает корзину (удаляет все товары), параметры: отсутствуют, возвращаемое значение: отсутствует (void).

getTotalPrice(): number - получение стоимости всех товаров в корзине, параметры: отсутствуют, возвращаемое значение: number — сумма цен всех товаров в корзине.

getItemCount(): number - получение количества товаров в корзине, параметры: отсутствуют, возвращаемое значение: number — количество товаров.

hasItem(id: string): boolean - проверка наличия товара в корзине по его id, полученного в параметр метода, параметры: id (string) — идентификатор товара, возвращаемое значение: boolean — true, если товар есть в корзине, иначе false.

Класс Buyer - хранение и управление данными покупателя.

Поля:
payment: TPayment | null; - способ оплаты товара  
email: string;- почта покупателя
phone: string;- телефонный номер покупателя
address: string; - адрес проживания покупателя

Методы Buyer:

Методы для сохранения данных в модели, отдельные для каждого поля.
setEmail(email: string): void
setPhone(phone: string): void
setAddress(address: string): void
setPayment(payment: TPayment): void

getData(): IBuyer | null - получение всех данных покупателя, параметры: отсутствуют, возвращаемое значение: IBuyer | null — полный объект данных покупателя или null, если данные неполные.

clearBuyer(): void - очистка данных покупателя, параметры: отсутствуют, возвращаемое значение: отсутствует (void).

validate() - проверяет обязательность заполнения полей покупателя. Возвращает объект с сообщениями об ошибках для невалидных полей, отсутствие поля означает, что оно прошло валидацию.

###### Слой коммуникации

interface IProductsResponse {// ответ сервера при запросе списка товаров или каталога
total: number; // общее количество товаров в базе
items: IProduct[]; // массив товаров, которые вернулись в ответе
}

interface IOrderData extends IBuyer { // данные для создания заказа (то, что отправляется на сервер)
total: number; // итоговая сумма товара при оплате в корзине у клиента
items: string[]; // Наследует от IBuyer, то есть включает все поля покупателя (email, phone, address, payment)
}

interface IOrderResponse { // ответ сервера после создания заказа
id: string; // уникальный идентификатор заказа, которая присваивается сервером
total: number; // итоговая сумма заказа, может быть пересчитана сервером для точности
}

Class ApiCommunication- представитель коммуникационного слоя — он будет отвечать за получение данных с сервера и отправку данных на сервер.

Поля:
api: IApi; // применяем принцип композиции, при котором один обьект использует другой обьект, в нашем случае, используем IApi в ApiCommunication

Методы:

1. getApi(): Promise<IProductsResponse> - получает данные из сервера
2. postApi(data: IOrderData): Promise<IOrderResponse> - отправляет данные на сервер и получает ответ
